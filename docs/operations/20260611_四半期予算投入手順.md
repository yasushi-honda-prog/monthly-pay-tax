# 四半期×統括隊×カテゴリ予算 投入手順 (PR-E)

予実管理機能の Phase 2 (四半期粒度 + 統括隊レイヤー + 7 支出カテゴリ) 用の予算投入手順。
既存 PR-A の月別予算 (`team_budgets`) は維持しつつ、別軸で `team_budgets_quarterly` に投入する。

## 会計年度の定義 (案 N11、11 月始まり)

| 四半期 | 月 |
|--------|-----|
| Q1 | 11月 / 12月 / 1月 |
| Q2 | 2月 / 3月 / 4月 |
| Q3 | 5月 / 6月 / 7月 |
| Q4 | 8月 / 9月 / 10月 |

`fiscal_year` は終了月 (10月) の暦年で揃える:
- 2025-11 〜 2026-10 が `fiscal_year=2026`
- 2026-11 〜 2027-10 が `fiscal_year=2027`

## 投入対象のテーブル

| テーブル | 内容 | 投入元 |
|---|---|---|
| `expense_categories` | 支出カテゴリマスタ (7 行) | migration SQL の seed (再投入不要) |
| `team_hierarchy` | 隊 ↔ 統括隊マッピング | `scripts/upload_team_hierarchy.py` |
| `team_budgets_quarterly` | 四半期 × 統括隊 × カテゴリの予算 | `scripts/upload_team_budgets_quarterly.py` |

## 手順 1: 階層マスタの投入 (初回 + 組織変更時のみ)

`docs/operations/team-hierarchy-template.csv` を編集 → 投入。

```bash
# dry-run でプレビュー + UNMAPPED/UNUSED 検出
python3 scripts/upload_team_hierarchy.py \
  docs/operations/team-hierarchy-template.csv \
  --dry-run --check-coverage

# 問題なければ本実行
python3 scripts/upload_team_hierarchy.py \
  docs/operations/team-hierarchy-template.csv \
  --check-coverage
```

`v_team_hierarchy_coverage` で `UNMAPPED` が出たら、その隊 (`activity_category`) を CSV に追加して再実行。

## 手順 2: 四半期予算の投入 (Q ごと)

### Matrix 形式 (画像を手で起こす場合、推奨)

`docs/operations/team-budgets-quarterly-template-matrix.csv` を編集 → 投入。

```bash
# Q3 (5-7月) 2026 の予算を投入
python3 scripts/upload_team_budgets_quarterly.py \
  docs/operations/team-budgets-quarterly-template-matrix.csv \
  --fiscal-year 2026 --fiscal-quarter 3 \
  --expected-total 23457444 \
  --dry-run

# 検算 OK なら本実行
python3 scripts/upload_team_budgets_quarterly.py \
  docs/operations/team-budgets-quarterly-template-matrix.csv \
  --fiscal-year 2026 --fiscal-quarter 3
```

`--expected-total` で画像合計値と検算 (不一致は warn のみ、本番投入は止めない)。

### Long 形式 (システム連携・スプレッドシート CSV 直接 export)

```bash
python3 scripts/upload_team_budgets_quarterly.py \
  budgets_long.csv \
  --expected-total 23457444 \
  --dry-run
```

Long 形式は `fiscal_year`, `fiscal_quarter` を CSV 内で指定するため `--fiscal-year` `--fiscal-quarter` option は不要。

## 確認クエリ

```sql
-- Phase 1 対応の予実 (タダメン業務委託費のみ)
SELECT
  fiscal_year, fiscal_quarter, leader_team, expense_category,
  budget_amount, actual_amount, achievement_rate, actual_mapping_status
FROM `monthly-pay-tax.pay_reports.v_team_budget_actuals_quarterly`
WHERE fiscal_year = 2026 AND fiscal_quarter = 3
  AND actual_mapping_status = 'mapped'
ORDER BY leader_team;

-- Phase 2 待ちのカテゴリ (予算のみ表示)
SELECT
  fiscal_year, fiscal_quarter, leader_team, expense_category, budget_amount
FROM `monthly-pay-tax.pay_reports.v_team_budget_actuals_quarterly`
WHERE fiscal_year = 2026 AND fiscal_quarter = 3
  AND actual_mapping_status = 'not_supported_in_phase1'
ORDER BY leader_team, expense_category;

-- 未マッピング隊の確認
SELECT activity_category, status
FROM `monthly-pay-tax.pay_reports.v_team_hierarchy_coverage`
WHERE status = 'UNMAPPED'
ORDER BY activity_category;
```

## 注意・運用上のルール

- 既存 PR-A の月別予算 (`team_budgets`) には影響なし。本機能は別テーブル `team_budgets_quarterly` 専用。
- 実額紐付けは Phase 1 では「タダメン業務委託費」のみ。他 6 カテゴリは `actual_mapping_status='not_supported_in_phase1'` で予算のみ表示される。
- `team_hierarchy` で `leader_team_type='operating'` の隊のみが gyomu_actuals に集計対象。`common` は予算用の virtual 統括隊で実額紐付けなし。
- snapshot バックアップは本 PR で `pay-collector` の `BQ_SNAPSHOT_TABLES` に追加済 (毎朝バッチ Step0 で 90 日 snapshot を取得)。
- **空セル / `-` / `—` は「未指定」として skip**。予算を 0 に戻したい場合は明示的に `0` を CSV に記載してください。空セルで「0 reset」を期待すると既存 BQ 行が残ります。
- **Excel 集計行 (計 / 合計 / 総計 / 小計 / total / sum / subtotal) は 1 列目に書けば自動 skip**。それ以外のラベル (例: 「組織全体」など) を集計行として使う場合は CSV から削除してください。
- **検算オプション `--expected-total` を強く推奨**: 画像の合計値と照合できる場合は必ず指定。デフォルトで不一致時は abort、`--allow-total-mismatch` で継続。
- **`gyomu_reports.year` 異常値や `extract_month` NULL の行は VIEW で silently filter される**ため、実額が想定より少ない場合は `v_team_hierarchy_coverage` の UNMAPPED + Step0 snapshot + `SELECT COUNT(*) FROM gyomu_reports WHERE SAFE_CAST(year AS INT64) IS NULL` を組み合わせて切り分け。

## エッジケース・失敗時のリカバリ

- **部分失敗 (lock 競合)**: 1 行ずつ MERGE しているため、ある行で lock 競合 → そのまま継続。完了サマリの「lock競合スキップ N 件」を確認し、必要なら `--force` で再投入。
- **部分失敗 (BQ error)**: 同上。`FAILED` 行を sed で抽出し、CSV を絞って再投入。
- **expected_total ミス**: `--allow-total-mismatch` で投入後に dashboard で確認、ズレを CSV に反映して再投入 (MERGE なので冪等)。
- **categories typo**: `expense_categories` マスタにないカテゴリは error 終了。migration SQL の seed に追加してから再投入。
