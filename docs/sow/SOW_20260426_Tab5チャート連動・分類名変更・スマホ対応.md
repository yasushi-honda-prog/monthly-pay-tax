## 1. エグゼクティブ・サマリー (Executive Summary)

Tab5（業務委託費分析）のドリルダウン操作性向上として、チャートバークリックとセレクトボックスの双方向連動を実装した。あわせて電話対応の分類名をケアプー実態に合わせて修正し、スマートフォンでグラフが表示されない問題をPlotly＋レスポンシブCSS対応で解決した。

---

## 2. プロジェクトの目的と背景 (Objectives & Background)

### 2.1 背景
Tab5の業務委託費分析チャートはバークリック→ドリルダウン機能を持つが、クリックとセレクトボックスが独立しており状態が不一致になるケースがあった。また「電話対応（主に行政事業中心）」という分類名が実態（ケアプランデータ連携システムを広め隊が主体）と乖離していた。さらにスマートフォンでは Altair チャートが全く表示されないという使用上の問題が報告された。

### 2.2 目的
- チャートクリックとセレクトボックスを単一ソースで連動させ、ドリルダウン操作を直感的にする
- 電話対応の分類名をケアプー実態に合わせた正確な名称に修正する
- スマートフォンでのグラフ表示・ドリルダウン操作を可能にする

---

## 3. 実施内容詳細 (Technical Scope of Work)

### 3.1 チャートクリック → セレクトボックス連動（ドリルダウン改善）

| 項目 | 内容 |
|------|------|
| 対象 | Tab5 `_render_cost_chart` 関数 |
| 変更内容 | バークリック時に `st.session_state[_sb_key]` を更新しセレクトボックスと同期。ダブルクリック解除で「選択なし」に戻る。`_last_chart_sel_key` で前回選択を追跡し再同期ループを防止 |
| 削除 | 旧実装（チャート選択を直接 `_selected_cost` に使う `else` ブランチ）を削除 |
| リセット対応 | リセット・選択解除ボタン押下時に `_last_chart_sel_key` もクリア |

### 3.2 電話対応の分類名変更

| 項目 | 内容 |
|------|------|
| 変更前 | `電話対応（主に行政事業中心）` |
| 変更後 | `電話対応（主にケアプランデータ連携システムを広め隊）` |
| 対象箇所 | `_COST_GROUP_MAP`（4キー）・`_COST_GROUP_EXCLUDE_NONPROFIT`（1件）・`prototype_cost_analysis.py`（5箇所） |
| 色ドメイン | `_COST_COLOR_DOMAIN` は `_COST_GROUP_MAP.values()` から自動生成のため変更不要 |

### 3.3 スマートフォン対応（Plotly + レスポンシブCSS）

| 項目 | 内容 |
|------|------|
| モバイル検出 | `st.context.headers.get("User-Agent")` でスマホ判定（`_is_mobile`） |
| Tab5 モバイル版 | `_render_cost_chart_mobile` 関数を新規追加。Plotly Express 積み上げ棒グラフ＋`on_select="rerun"`でタップドリルダウン対応 |
| タップ連動 | バータップ → `legendgroup` 取得 → セレクトボックス連動（Altair版と同方式） |
| ドリルダウン | Plotlyメンバー別積み上げ棒＋メンバー合計テーブル（シンプル版） |
| レスポンシブCSS | `@media (max-width: 767px)` で `wide` レイアウトの横スクロール防止・パディング調整 |
| 依存追加 | `plotly>=5.0.0` を `requirements.txt` に追加 |

## 6. 今後の推奨事項 (Recommendations)

- Tab1（月別報酬サマリー）・Tab2（スポンサー別）のチャートも Altair を使用しており、モバイルで表示されない可能性がある。必要に応じて同様の Plotly 条件分岐対応を検討
- Plotly `on_select` の `legendgroup` 取得は実機タップで動作確認が必要。想定外のデータ構造の場合はセレクトボックス選択にフォールバックされる設計のため実用上の問題は低い
- `_is_mobile` はモジュール起動時に一度評価されるため、同一セッション内でデバイスを切り替えても再検出されない（再ロードで解決）
