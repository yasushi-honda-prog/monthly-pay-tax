# 作業報告書（SOW）

プロジェクト: タダカヨ 活動時間・報酬マネジメントダッシュボード
対象システム: pay-dashboard（Cloud Run / Streamlit）
作業期間: 2026年3月14日（土）〜 3月15日（日）
作業者: Claude Code（AI開発支援）


---

## 作業概要

Streamlitダッシュボードのユーザビリティ改善を実施。UIの視認性・操作性の向上、データ集計・フィルタリング機能の拡充、バグ修正を行い、Cloud Runへデプロイした。

---

## 実施内容

### 1. 機能追加（3/14）

| 項目 | 内容 |
|------|------|
| グループ選択プルダウン | サイドバーにグループ選択を追加。グループでメンバーリストを絞り込み可能に |
| 業務分類フィルター（複数選択） | 業務報告一覧タブの業務分類フィルターを複数同時選択対応に変更 |
| KPIカード追加 | 業務報告一覧タブ・グループ別タブの業務報告サブタブに総額・件数・メンバー数のKPIカードを追加 |

---

### 2. 期間指定フィルター実装（3/15）

| 項目 | 内容 |
|------|------|
| 「期間指定」モード追加 | 月選択の「全月」を「期間指定」にリネームし、開始年月・終了年月を自由に指定できるUIを実装 |
| 年またぎ対応 | 例: 2025年11月〜2026年10月など会計年度をまたぐ期間指定に対応 |
| デフォルト期間 | 決算月（10月）を基準に当年度の11月〜翌年10月を自動設定 |
| 選択UI改善 | 開始・終了を「YYYY年M月」形式の単一プルダウン1つに統合し、幅の問題を解消 |
| 前月デフォルト | 特定月指定時のデフォルトを「本日の前月」に変更（例: 3月ならば2月を選択） |

---

### 3. データ集計・表示改善（3/15）

| 項目 | 内容 |
|------|------|
| 年月表記の統一 | 月次推移グラフ・スポンサー別ピボットの列見出しを数値から「YYYY年M月」形式に変更 |
| グループ選択時の自動全選択 | サイドバーでグループを選ぶと、そのグループのメンバーが自動的に全選択される |
| スポンサー別ピボット 期間フィルター | 期間指定がスポンサー別業務委託費タブに反映されていなかった不具合を修正 |
| 空白月を0表示 | スポンサー別ピボットで期間内にデータがない月も0として列表示（分析の一貫性確保） |
| 不正データ除去 | 月データが空のレコードが「2026年0月」として表示されていた不具合を修正 |

---

### 4. グループ別タブ拡充（3/15）

| 項目 | 内容 |
|------|------|
| サイドバーフィルター連携 | グループ別タブでもサイドバーのメンバー選択・期間指定が反映されるよう改修。グループ選択×個人選択の組み合わせ分析が可能に |
| 期間指定対応 | グループ別タブの月別報酬・業務報告で期間指定フィルターが正しく動作するよう修正 |
| ピボット形式統一 | グループ別の月別報酬ピボットを「YYYY年M月」形式に統一 |
| グループ戻り時バグ修正 | 特定グループから「全グループ」に戻した際にメンバーチェックがリセットされず、フィルターが残り続ける不具合を修正 |

---

### 5. UI・デザイン改善（3/14）

| 項目 | 内容 |
|------|------|
| サイドバーナビゲーション | ページリンクのフォントサイズを拡大（視認性向上） |
| ページ上部余白 | ヘッダー上の余白を縮小し、コンテンツ表示領域を拡大 |
| サイドバーフォント統一 | ラベル・入力値・メンバーリストのフォントサイズを統一 |
| メンバーリスト | 行間を詰めてスクロールなしで表示できる件数を増加。高さを460pxに調整 |
| サイドバーセクション | 「期間」タイトルを削除し、余白を整理 |
