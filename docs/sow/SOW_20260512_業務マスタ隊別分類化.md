## 1. エグゼクティブ・サマリー (Executive Summary)

業務マスタの隊別分類化に伴い、業務分類提案CSV（`docs/gyomu_master_proposal.csv`）の文言修正・行追加を実施した。あわせて隊長レビュー用Googleスプレッドシートの構成・書式設計、および配布・回収フローを策定した。

---

## 2. プロジェクトの目的と背景 (Objectives & Background)

### 2.1 背景

従来の業務分類マスタは「タダカヨ業務全般」「関連チーム業務」など組織横断的な表現を使用していた。隊別分類化の方針決定に伴い、各隊固有の業務分類へ再整理する必要が生じた。

### 2.2 目的

- 業務分類提案CSVを隊別分類化の方針に合わせた内容へ修正する
- 隊長がレビュー・意見入力できるスプレッドシートを整備し、合意形成プロセスを設計する

---

## 3. 実施内容詳細 (Technical Scope of Work)

### 3.1 業務分類マスタ提案CSV の修正

| 項目 | 内容 |
|------|------|
| 業務内容（例）の文言修正 | 「タダカヨ業務全般にかかる移動時間」→ 各隊固有の表現へ変更（出張タダスク・AI探検隊・ケアプー・神奈川県事業の4件） |
| 「タダスク関連」の業務内容修正 | 「新規メンバー対応 / 関連チーム業務（みんなでスキルアップ隊・介護DXで包括の未来を応援し隊）」→「新規メンバー対応」のみへ整理 |
| 「しっかり法人を経営し隊」の業務内容修正 | 末尾の「/ 関連チーム業務」を削除し隊固有業務のみ記載 |
| 新規行の追加 | みんなでスキルアップし隊・介護DXで包括の未来を応援し隊に「新規メンバー対応」業務（単価3,000円）を独立追加 |

### 3.2 隊長レビュー用スプレッドシートの設計

| 項目 | 内容 |
|------|------|
| スプレッドシート構成 | A:活動分類（隊）/ B:業務分類 / C:業務内容（例）/ D:業務単価 / E:ステータス / F:意見・修正案（6列） |
| ヘッダー書式 | 濃いグレー背景（#434343）・白太字・行固定 |
| 隊別色分け | 15隊それぞれ異なるパステルカラー（Apps Script で適用） |
| 追加希望行 | 各隊末尾に2行（グレー斜体・「追加」ステータス設定済み） |
| データ検証 | E列にドロップダウン（承認／修正／削除／追加） |
| 行数 | 106行（ヘッダー1行 + データ105行、うち追加希望行30行） |

### 3.3 配布・回収フローの設計

| 項目 | 内容 |
|------|------|
| 配布方式 | スプレッドシートを隊長全員に編集権限で共有（A案採用） |
| 通知手段 | SlackまたはメールでURLと締切日を通知 |
| 回収後作業 | ステータス確認→`report_input.py` の `GYOMU_MASTER` と `dashboard.py` の `_COST_GROUP_MAP` を更新→Cloud Run デプロイ |
