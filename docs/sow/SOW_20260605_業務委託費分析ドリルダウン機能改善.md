## 1. エグゼクティブ・サマリー (Executive Summary)

業務委託費分析タブのドリルダウン機能を強化した。業務分類別内訳の表に複数行選択フィルタを追加し、選択した業務分類に絞り込んで分類合計・メンバー数・グラフを確認できるようにした。あわせて表の円表記バグを修正し、グラフ・凡例の隊の並び順を業務別報酬単価表のスプレッドシート順に変更した。

---

## 2. プロジェクトの目的と背景 (Objectives & Background)

### 2.1 背景
隊を選択してドリルダウンを開いた際、業務分類別内訳の表が表示専用で、特定の業務分類を選択してもKPI（分類合計・メンバー数）やグラフが絞り込まれない問題があった。また隊の並び順がアルファベット順になっており、スプレッドシートとの対応がとれていなかった。

### 2.2 目的
- 業務分類別内訳の表で行を選択し、KPI・グラフをその業務分類に絞り込めるようにする
- 複数業務分類の同時選択・合算表示を可能にする
- グラフ・凡例の隊の並び順をスプレッドシートに合わせる

---

## 3. 実施内容詳細 (Technical Scope of Work)

### 3.1 業務分類別内訳 行選択フィルタ追加（PR #168）

| 項目 | 内容 |
|------|------|
| 変更前 | 表は表示専用、KPI・グラフは常に隊全体の合計 |
| 変更後 | 行を選択（複数可）するとKPI・グラフが選択した業務分類に絞り込まれる |
| 対応 | PC版（_render_cost_chart）・モバイル版（_render_cost_chart_mobile）両方 |

### 3.2 円表記バグ修正・複数選択対応（PR #169）

| 項目 | 内容 |
|------|------|
| 不具合 | `column_config.NumberColumn(format="¥{:,.0f}")` がリテラル表示されていた |
| 修正 | `style.format()` に変更して正しく `¥1,000` 形式で表示 |
| 追加 | `selection_mode="multi-row"` で複数業務分類の同時選択が可能に |

### 3.3 on_select 選択不具合修正（PR #170）

| 項目 | 内容 |
|------|------|
| 不具合 | Stylerオブジェクトを渡すとon_selectが動作しないStreamlitの制約 |
| 修正 | `apply(lambda x: f"¥{x:,.0f}")` で事前フォーマット済み文字列列に変換してから渡す |

### 3.4 隊分類の並び順をスプレッドシート順に変更（PR #171）

| 項目 | 内容 |
|------|------|
| 変更前 | `sorted()` による五十音順 |
| 変更後 | 業務別報酬単価表（gid=700881857）の隊分類の並び順に一致 |
| 対象 | グラフの積み上げ順・凡例・ピボット表の行順 |
