## 作業概要

ダッシュボードの安定性回復・パフォーマンス最適化・業務チェックページの機能拡充・UI改善を実施し、Cloud Runへデプロイした。

---

## 実施内容

### 1. 安定性回復・パフォーマンス最適化

| 項目 | 内容 |
|------|------|
| 不安定なパフォーマンス改善をリバート | グループ処理リファクタリングが挙動不安定の原因となったため、モバイル修正後の安定状態に戻した |
| TTL延長のみ再適用 | BQクエリキャッシュのTTLを1時間→6時間に延長（dashboard.py全関数・bq_client.py）。グループ処理リファクタリングは含まない安全な最適化のみ実施 |

---

### 2. 業務チェックページ — 手動データ更新ボタン

| 項目 | 内容 |
|------|------|
| 手動更新ボタン追加 | 業務チェックページ最下部にexpander「データ更新（手動）」を追加。checker/adminが利用可能 |
| Collector呼び出し | ボタン押下でCloud Run CollectorにOIDC認証付きPOSTリクエストを送信 |
| 完了待機型 | 約4分のスピナー表示で完了まで待機し、成功/失敗を画面に表示 |
| 目立たない設置 | 操作ログの下にexpander（折りたたみ）形式で設置 |
| 定数・依存追加 | lib/constants.py に COLLECTOR_URL を追加。requirements.txt に requests を明示追加 |

---

### 3. 業務チェックページ — 操作ログ改善

| 項目 | 内容 |
|------|------|
| デフォルト選択を最終保存者に変更 | 操作ログのメンバー選択を、最も直近にステータス変更・メモ保存を行ったメンバーに自動フォーカス |
| ログなし時は未選択表示 | 当月まだ誰も操作していない場合は「選択してください」と表示し任意選択を促す |
| 2行表示バグ修正 | 未選択時に「操作ログはありません」が2行表示されていたバグを修正 |

---

### 4. ダッシュボード — 件数・人数バッジ追加

| 項目 | 内容 |
|------|------|
| 人数バッジ（メンバー別 月次支払額） | ピボットテーブルのタイトル右に「○ 名」バッジをインライン表示 |
| 件数バッジ（メンバー別 報酬明細） | 明細テーブルのタイトル右に「○ 件」バッジをインライン表示 |
| インライン配置 | flexboxレイアウトでタイトルとバッジを横並び表示。既存の count-badge CSSスタイルを活用 |
