### 2. Google スプレッドシートの整備

**スプレッドシート名**: 業務分類マスタ（隊別）提案  
**URL**: https://docs.google.com/spreadsheets/d/1-raP8DBvI3W-Pmsv673vKbQklMBh3y8X5fjj2Uj_qew/

#### 構成

| 列 | 項目 | 役割 |
|---|---|---|
| A | 活動分類（隊） | 所属隊（15分類） |
| B | 業務分類 | 業務ドロップダウン名（一意） |
| C | 業務内容（例） | 具体的な作業内容 |
| D | 業務単価（円/h） | 単価（既存単価体系から選択） |
| E | ステータス | 承認 / 修正 / 削除 / 追加（ドロップダウン） |
| F | 意見・修正案 | 隊長による自由記入欄 |

#### 書式設定内容（Apps Script で適用）

- **ヘッダー行**: 濃いグレー背景（#434343）・白太字・行固定
- **隊別色分け**: 15隊それぞれ異なるパステルカラー
- **追加希望行**: 各隊末尾に2行（グレー斜体・「追加」ステータス設定済み）
- **列幅調整**: 各列を内容に応じて最適化
- **データ検証**: E列にドロップダウン（承認/修正/削除/追加）

**行数**: 106行（ヘッダー1行 + データ105行）  
**うち追加希望行**: 30行（各隊2行 × 15隊）

### 3. 配布・回収方法の設計

#### 採用方式: スプレッドシート意見列方式（A案）

**配布手順**:
1. スプレッドシートを隊長全員に**編集権限**で共有
2. SlackまたはメールでURLと締切日を通知
3. 依頼文の要点:
   - 自分の隊の行を確認し、E列（ステータス）を入力する
   - 修正・削除は F列（意見・修正案）に詳細を記入する
   - 新規追加希望は各隊末尾の追加行（グレー）に記入する
   - 単価は既存単価リストから選ぶ（30/1000/1500/2000/2500/3000/3500/4000/5000/6000/7000/15000/30000）
   - 業務分類名は「隊略称_業務名」形式で記入（重複防止）

**回収後の作業**:
- 全隊のステータスを確認し、修正・追加を反映
- `report_input.py` の `GYOMU_MASTER` および `dashboard.py` の `_COST_GROUP_MAP` を更新
- 本番環境への反映（Cloud Run デプロイ）
