## 3. 配布ツールの仕組み

`立替金等入力シート配布作業範囲記述書` の内容（2025-12-24 作成、作成者: ゆりさん＝近藤ゆり）より:

### 3.1 Step 1: 配布状況調査システム

- **ソースフォルダ**: 業務報告シートフォルダ（ID: `1iwXRFv4OJkKeIC2TEPZpndnxp_0YjDbt`）
- **調査対象タブ**: 各業務報告シート内の `【月１入力】補助＆立替報告＋月締め` タブ
- **ロジック**:
  1. ファイル名から正規表現 `/【(.*?)】/` でニックネームを抽出
  2. セル K3 の値を抽出
  3. K列・L列を全行走査し、キーワード「立替金等入力シート」を含む最初のセル番地を特定
  4. ファイル名命名規則（`【ニックネーム】委託事業_立替金等入力シート_仮払い機能付`）でドライブ全体を検索
  5. 結果を配布リストに出力

### 3.2 Step 2: 自動配布システム

- **データソース**: アクティブスプレッドシートの第1シート（管理リスト）
- **対象フィルタリング**: ステータス「未」かつ「②」を含まないメンバー
- **処理**:
  1. テンプレート複製 → ファイル名整形（`【ニックネーム】` + 規定の接尾辞）
  2. 業務報告シートの編集者・閲覧者権限を新規シートへ継承
  3. 業務報告シートの `【月１入力】補助＆立替報告＋月締め` タブの **K3セル** にURLを設置
  4. スマートチップ化を促すメモを添付

## 6. 残る技術的懸念

1. **セル番地のばらつき**: 業務報告シートのK3/L3/K6/K7/K8/K12/L4/L13 に立替金シートURLが埋め込まれている（メンバー別）。collector実装時に配布リストを参照する必要あり
2. **スマートチップ形式**: 配布作業範囲記述書に「スマートチップ化されている場合、検出精度が低下する」との記述あり。Sheets API 経由でのURL取得方式を検証要
3. **仮払い機能の意味論**: 「仮払い機能付」シートの仮払金額列はWAMフロー上どう扱うか（経費精算済 vs 未精算の区別）
4. **「その他」対象PJの雑多運用**: 現状の「その他」にはオフ会・佐伯PJ・大分ナガヨシ展示会など雑多な事業が混在。WAMは専用PJ名（例: `WAM-出張タダスクPJ`）を新設すべき
5. **ファイル名の一意性保証**: Step1記述書に「同名ファイル複数時は最初の1つ」との制約あり。ドライブ検索よりも配布リストからの直接ID参照が安全
6. **実行時間**: Step1 GAS記述書に「1回の実行で100件程度まで」との記述あり。現在の pay-collector の Cloud Run 実行（約217秒）に立替金巡回を加えた場合の処理時間を要検証
