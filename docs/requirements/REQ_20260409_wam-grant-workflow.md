## 2. 前提：現状の入力シート構成

以下の2つのシートが入力元として存在しており、今回の改修はこれらをベースとする。**入力側の運用（タダメンの操作）は変えない方針**。

| シート | 内容 |
|--------|------|
| 業務報告シート | 業務委託費（報酬）を記録するシート。活動分類の項目あり。 |
| 立替えシート | 旅費・経費の立替分を記録するシート。WAM対象フラグ列の追加を想定。 |

### WAM対象の識別方法（裏側ロジック）

- **業務報告シート**: 「活動分類（または業務分類）」の項目を事前に定義し、対象レコードを自動抽出する。「出張タダスク（WAM対象）」など専用の活動分類をプルダウンに追加する可能性あり。
- **立替えシート**: 「WAM助成金対象」列（チェックボックス）を新たに追加し、フラグが立ったレコードを抽出・集計する。

## 3. 要件詳細

### ① 領収書の自動生成と振込一括化【Must / 要相談】

#### 背景・目的

振込担当の手数料負担軽減と、タダメンの印刷・郵送コスト削減のため、**業務委託費と旅費（立替金）を合算して一括振込できる運用**を目指したい。

#### 要件

以下2パターンの領収書PDF（または同等の出力）を自動生成または選択出力できるようにしたい。

**【パターン1】**
- A: 業務委託費のみの領収書 → 1振込
- B: 旅費（立替金）のみの領収書 → 1振込

**【パターン2】**
- C: 上記2つを合算した総額（A＋B）の領収書（事務局の理想形） → 1振込

▶ **事務局の希望はパターンCの運用**。ただし実現が難しい場合はA+Bの2振込でも許容できるか検討する。

#### 技術的な確認事項（ヤススさんへ）

- 「業務報告シート（委託費）」と「立替えシート（旅費）」は現在入力元が別々だが、両シートのデータを紐づけて1枚のPDFに落とし込む実装は可能か？
- それぞれのシートから個別に最終振込処理を行う方法も選択肢として現実的か？

### ② 振込データの抽出ロジックと一括振込CSV出力【Must / 要相談】

#### 背景・目的

現在PayPay銀行の一括振込ツールを運用中。WAM助成金事業用を同ツール上で別系統として管理したい。

> ※ PayPay銀行でのWAM専用口座の複数作成可否についてはヒロスさんが確認 → **結果: 作成不可**。WAM専用口座は「**GMOあおぞらネット銀行**」で開設する方針に確定。

#### 要件

- 既存の報告シートのフラグ（WAM対象 / 非対象）に基づき、振込データを「通常口座用」と「WAM専用口座用」の2系統に自動振り分ける。
- 既存のPayPay銀行一括振込ツールのWAM版を作成し、そこからCSVを一括出力できるインターフェースを構築する。

#### 技術的な確認事項

- 上記の2系統振り分け＋CSV一括出力インターフェースの構築は可能か？
- 既存の振込ツールの改修で対応できるか、新規作成が必要か？

#### ヒデスさんコメント

> 一括振込やるなら、paypay銀行と違うフォーマットで出力しないといけないですね。
> https://gmo-aozora.com/support/paymentguide.html

### ③ WAM月別報酬・源泉徴収確認ツール【Must（表はWant）/ 要相談】

#### 背景・目的

事業全体の支出状況の把握と、経理担当が振込前に確認作業を行う際の参考データとして使用する。

#### 要件

- WAM助成金事業に係る業務委託費・経費の総合計を月別に一覧で確認できること。
- 業務報告シートから抽出した各レコードを、項目ごとに整理して表示できること。
- 書式はスプレッドシートに限らず、閲覧・チェックがしやすい形であれば可。

#### 技術的な確認事項（ヤススさんへ）

- 既存の業務報告シートからの抽出ロジックを流用する形で実装可能か？
- 独立したツールとして作成するか、既存シートの集計タブとして実装するか、どちらが望ましいか意見を聞きたい。

### ④ 支払調書作成ツールへの連携【Want / 要相談】

#### 背景・目的

WAM助成金事業分の報酬・委託費を既存の支払調書作成ツールと合算し、対象者（対象法人）に対して1枚の支払調書を出力したい。

#### 技術的な確認事項

- 既存の支払調書作成ツールに、WAM分のデータを合算して取り込む連携は実装可能か？
- 合算の対象者・対象法人の識別ロジックについて、現状ツールの構成を踏まえてアドバイスが欲しい。

### Phase 1b: 立替金シート側のデータ基盤 🆕

**`REF_20260411` の調査結果により Phase 1a と並行着手可能に昇格**

- **既存の分析系（gyomu_reports / hojo_reports / 既存VIEW / 既存ダッシュボード）は無変更**

### Phase 2: 月別確認ツール（要件③ Must）

- 新規VIEW `v_wam_monthly_summary`（業務報告側+立替金側をUNION、月別集計、源泉徴収計算）
- ダッシュボードに新ページ `pages/wam_summary.py`（または既存dashboard.pyに新タブ）
- アクセス制御: checker / admin / 経理担当
- 独立ツール vs 既存ダッシュボード統合 → **既存ダッシュボードに新ページ追加を推奨**（認証・BQ接続・UI基盤を流用）

### Phase 3: 領収書生成（要件① Must）

- **第一案: 既存PDFの集約アプローチ**（立替金シートの `receipt_url` 列を既に保有しているため新規PDF生成が不要な可能性）
- 第二案: 業務委託費＋立替金を合算した統合領収書PDFを新規生成（`weasyprint` 等）
- ダッシュボードに「領収書出力」機能を追加
- 押印フローは人手運用を維持（前回の三文判代理押印は廃止）

### Phase 4: 振込CSV出力（要件② Must）

- GMOあおぞらネット銀行フォーマット対応（PayPay版と別実装）
- ダッシュボードに「振込CSV出力」機能を追加
- Phase 3 と独立実装可能だが、同じデータソース（`v_wam_monthly_summary`）を使用

### Phase 5: 支払調書連携（要件④ Want）

- 既存「支払調書作成ツール」の現物確認後に対応方針を決定
- Phase 1〜4の安定後に着手
